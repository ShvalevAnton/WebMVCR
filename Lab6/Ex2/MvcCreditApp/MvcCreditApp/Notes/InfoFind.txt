1)Почему используется поиск FindAsync "var bid = await _context.Bids.FindAsync(id);", а не как методе Details "var bid = await _context.Bids.FirstOrDefaultAsync(m => m.BidId == id);"

Основные различия:

___Механизм поиска:
FindAsync(): Ищет сначала в кеше отслеживаемых сущностей (локальном кеше DbContext)
Только если не находит - выполняет запрос к базе данных Оптимизирован специально для поиска по первичному ключу.
FirstOrDefaultAsync():
Всегда выполняет запрос к базе данных. Не проверяет локальный кеш перед выполнением запроса.

___Производительность:
FindAsync() обычно быстрее, если сущность уже загружена в контекст
FirstOrDefaultAsync() всегда делает запрос к БД, что может быть менее эффективно

___Гибкость:
FindAsync() работает только с первичным ключом
FirstOrDefaultAsync() может использовать любое условие (где, сортировки и т.д.)

___Почему в Edit используется FindAsync?
Специфика редактирования:
В Edit мы точно знаем, что ищем по первичному ключу (ID)
Высокая вероятность, что сущность уже могла быть загружена в контекст

___Оптимизация:
Если сущность уже была загружена (например, при предыдущих операциях), FindAsync вернет ее из кеша.
Это уменьшает количество обращений к БД.

___Когда что использовать?
FindAsync - когда точно ищем по PK и важна максимальная производительность
FirstOrDefaultAsync - когда нужно более сложное условие или важна читаемость кода
